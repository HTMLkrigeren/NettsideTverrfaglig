HVA ER JS?

JS står for JavaScript. Det er et programmeringsspråk som gjør nettsider levende. Mens HTML gir nettsiden innhold og CSS bestemmer stil og utseende, lar JavaScript deg lage interaktivitet. Med JS kan du få knapper til å reagere når du klikker, hente data fra andre steder, og oppdatere innhold uten å måtte laste siden på nytt.

HVORDAN FUNGERER JS?

JavaScript kjører i nettleseren din. Når du åpner en nettside, leser nettleseren HTML-en, CSS-en, og til slutt JS-koden. JS kan for eksempel lytte etter at du klikker på en knapp, og så kjøre en kode som viser en melding eller endrer tekst på skjermen. JS legger muskler på siden og gjør at du kan klikke, dra, skrive og se endringer uten å laste siden på nytt.

HVORFOR ER JS VIKTIG?

JavaScript er viktig fordi det gjør nettsider interaktive og brukervennlige. Uten JS ville alle nettsider vært statiske, du måtte laste siden på nytt for å se noe nytt. Med JS kan du lage ting som bildepresentasjoner, chat-vinduer, sjekke skjema før du sender det, og oppdatere innhold i sanntid. JS brukes også på server siden, så det er veldig allsidig. Det er en stor grunn til at moderne nettsider føles raske og dynamiske.

NOEN ELEMENTER SOM BRUKES INNENFOR JS:

  • DOM (Document Object Model):

    DOM er en måte nettleseren ser på nettsiden din som et tre av bokser. Hver boks er et HTML element, som for eksempel en < div >, < p > eller < button >. Med JavaScript kan du finne disse boksene og endre dem, for eksempel: bytte tekst, farge eller hva som skjer når du klikker på noe.

    Eksempel på DOM i bruk:
    Hvis du har < h1 id="tittel" >Hei< /h1 >; i HTML, kan du i JS skrive document.getElementById("tittel").textContent = "Ny overskrift"; for å endre teksten.

  • document.getElementById(#ID):

    ID er svært viktig når det gjelder JS, metoden i overskriften henter ett enkelt element fra DOM-en basert på verdi av ID. Hvis du gir et element id="btn"; i HTML, kan du i JS skrive: 

    
                        let knapp = document.getElementById("btn");

    Nå har du en variabel knapp som representerer akkurat det elementet og kan for eksempel legge til en konfigurere den som å endre stil og diverse.

  • addEventListener():

    Dette fester noe som heter en event listener til et DOM-element, slik at du kan kjøre en funksjon når noe skjer, for eksempel når brukeren klikker. 

    
                        let knapp = document.getElementById("btn");
                        knapp.addEventListener("click", function() {
                            alert("Knappen ble klikket!");
                        });

    Det betyr: når brukeren klikker på elementet med id="btn", kjører funksjonen inne i addEventListener.

  • if, else, if else (betingelser):

    Betingelser lar programmet sjekke om noe er sant eller usant, og kjøre forskjellige kodestykker via de betingelsene. 

    
                        let alder = 20;
                        if (alder >= 18) {
                            console.log("Du er voksen");
                        } else if (alder >= 13) {
                            console.log("Du er tenåring");
                        } else {
                            console.log("Du er et barn");
                        }

    Man sjekker først om alder er 18 eller mer. Hvis ikke, ser man om den er 13 eller mer. Hvis ingen av de er sanne, går man til "else" blokken.

  • for-loops:

    En for-løkke kjører samme kode flere ganger, så lenge en betingelse er sann. Brukes ofte for å gå gjennom en liste eller telle opp. 

    
                        for (let i = 0; i < 5; i++) {
                            console.log("Teller: " + i);
                        }

    Her starter i, som er normen når man bruker for-løkker, på 0. Så lenge i < 5 (i er mindre en 5), kjører koden inne i løkken og øker i med 1 hver gang. Tallene 0 til 4 vises da i konsollen.

  • while-loops:

    En while-løkke kjører så lenge en gitt betingelse er sann, den er veldig lik for-løkker, forskjellen fra for-løkker er at du selv må passe på å oppdatere telle variabelen inni løkken. 

    
                        let teller = 0;
                        while (teller < 3) {
                            console.log("Teller er nå: " + teller);
                            teller++; // øker teller med 1 hver gang
                        }

    Her starter teller på 0. Løkken kjører så lenge teller < 3, og øker teller med 1 hver gang. Den skriver 0, 1 og 2.

  • forEach (og andre løkker over arrays):

    Hvis du har en liste (array), en array ser sånn her ut [], med flere elementer, kan du bruke forEach for å kjøre en funksjon som går innom hvert enkelt element. 

    
                        let tall = [2, 4, 6];
                        tall.forEach(element => {
                            console.log(element * 2);
                        });

    Her tar forEach en og en oppføring i tall arrayen (2, 4, 6), kaller den element, og skriver element * 2 i konsollen (4, 8, 12).

  • API (Application Programming Interface):

    En API er en måte å hente data eller sende data til en ekstern tjeneste (server) på. Tenk på det som en sted der du kan be om informasjon: værdata, personinfo, eller bilder, uten å måtte vite alt om hvordan serveren bak fungerer.

    I JavaScript bruker man ofte fetch for å kontakte et API og be om data.

  • JSON Filer:

    JSON er et enkelt format for å sende data frem og tilbake mellom server og klient. Det ser ut som JavaScript-objekter, for eksempel: 

    
                        {
                            "navn": "Kari",
                            "alder": 25,
                            "yrke": "Utvikler"
                        }

    Når du har hentet en JSON-streng fra et API, bruker du metoden response.json() for å gjøre det om til et JavaScript objekt du kan jobbe med.

  • fetch():

    fetch er en innebygd funksjon i nettleseren som lar deg be om noe fra en server, for eksempel en API. Den returnerer et promise (løfte) som du kan bruke .then() og .catch() på. 

    
                        fetch("https://api.example.com/data")
                            .then(response => response.json()) // gjør om rådata til JS-objekt
                            .then(data => {
                                console.log(data); // jobber med det vi har hentet
                            })
                            .catch(error => {
                                console.error("Noe gikk galt:", error);
                            });

    Forklaring steg for steg:
    1. fetch("URL") sender en forespørsel til URL-en.
    2. Når serveren svarer, får du et response objekt. Men du kan ikke jobbe med det før du bruker response.json() som konverterer JSON-teksten til et JS objekt.
    3. Når du har JS objektet, kjører du den neste then blokken, hvor du kan bruke dataene.
    4. Hvis noe feiler (for eksempel nettverksfeil), havner du i catch blokken.

  • Arrow-funksjoner (=>):

    Arrow funksjoner er en kortere måte å skrive funksjoner på. Den erstatter ordet function med piler og fjerner behovet for return hvis du bare har en uttrykkslinje. 

    
                        // Tradisjonell funksjon:
                        function leggeSammen(a, b) {
                            return a + b;
                        }

                        // Arrow-funksjon:
                        let leggeSammen2 = (a, b) => a + b;

    Eksempel med forEach over en array: 

    
                        let tall = [1, 2, 3];
                        tall.forEach(t => console.log(t * 2));

    Her tar t som er hvert tall i arrayen og skriver t * 2 i konsollen.

  • API-håndtering med fetch og JSON, steg for steg:

    1. fetch("https://...") sender forespørselen.
    2. .then(response => response.json()) konverterer svaret til JSON.
    3. .then(data => { ... }) jobber med JSON-dataene som et JS-objekt/array.
    4. .catch(error => { ... }).catch fanger eventuelle feil (nettverksfeil, ugyldig JSON, osv.).

    Et eksempel: 

    
                        fetch("https://api.example.com/brukere")
                          .then(response => response.json())
                          .then(brukere => {
                            brukere.forEach(b => {
                              console.log(b.navn, b.alder);
                            });
                          })
                          .catch(err => {
                            console.error("Noe gikk galt:", err);
                          });
  • forEach vs for-løkke, hvem er best?:

    Både forEach og for-løkker kan brukes til å gå gjennom en liste [array]. forEach er ofte mer lesbart når du bare vil utføre en handling på hvert element, uten å måtte styre iterasjonen manuelt. Kanskje er det bare meg, men jeg liker forEach, fordi det er enklere å bruke. Det kan godt hende at det bare er en vanesak. 

                            // For-løkke:
                        let frukt = ["eple", "banan", "appelsin"];
                        for (let i = 0; i < frukt.length; i++) {
                            console.log(frukt[i]);
                        }
                        
                        // forEach:
                        frukt.forEach(f => console.log(f));

    Begge metodene skriver ut alle fruktnavnene, men forEach er kortere og enklere å lese.
